Todos alguna vez hemos tenido proyectos ordenados asi, por ejemplo:

    -portfolio-version-1 <- Armamos la estructura principal
    -portfolio-responsive <- Le dimos responsive
    -portfolio-final <- Esta el CSS terminado con animaciones

Git es un sistema de control de versiones
Esto, que termina siendo super enredado, git evita que lo hagamos.

Git permite:
-Guardar versiones
-Volver atras
-Trabajar en equipo
-No perder codigo

Concepto clave:
Git guarda "snapshots" del proyecto.
Imaginemos que git, podemos pedirle que "saque una foto" del proyecto, y que lo guarde, versionandolo


Local Y Remoto

Nosotros podemos tener una version del proyecto en nuestro local, y no estar en Remoto, y viceversa.
Puede haber cambios en el remoto que no tengamos en el local.

Si mi local tiene cambios que el remoto no tiene, esos cambios "solo" existen en mi pc.
Al yo actualizar el remoto con mis cambios, esos cambios pasan a ser del remoto, y estar accesible para todos.
